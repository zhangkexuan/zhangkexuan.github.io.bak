HTTP Status 404

最常见的404错误。直接查看请求：

属于请求不到资源，路径问题，可发现请求地址中本想用request来动态获取，却获取失败，以为request中没用值。改为正确地址即可。

HTTP Status 500 – Internal Server Error

问题已经可以看出了：500就是服务器出错了。提示说Servlet.init()抛异常。就先检查servlet是否设置正确，可以写个demo来测试servlet是否可以访问。

exception抛的异常不是关键，下面也有提醒：xml文件中无法正确解析springmvc.xml，说明springmvc.xml有问题。

下面又说了无法找到context:component-scan的声明，应该xmlns:context正确使用但xsi:schemaLocation的引用问题。
