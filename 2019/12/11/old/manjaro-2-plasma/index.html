manjaro#2-plasma

发表于 2019-12-11 分类于 其他技术

折腾，继续折腾……今天好不容易把manjaro的软件更新、内核升级搞定，现在我的plasma又崩溃了……我回忆了一下，我在添加桌面插件的时候及其容易崩溃，现在得看看到底哪儿出了错，该咋个解决……

kde 桌面环境不仅华丽而且有很多可以自定义的功能，可以组合配置出个性、实用、炫酷的桌面环境，虽然很多人评价kde的稳定性还不错，但是它仍然时不时的会崩溃。
遇到崩溃的时候可能有一些和我一样的新手只能选择reboot重新启动了，不过今天和大家分享一个重启plasma5的办法，也许能够帮到大家。

整体思路

想办法启动终端
杀掉或退出不正常的plasmashell
重启plasmashell

启动终端的方法

  1. 使用 ctrl + alt + t （或设置的快捷键）呼出终端
  2. 尝试 F12 启动 yakuake
  3. 使用tty Ctrl + Alt + Fx（F2 - F6，F1 是图形环境) （实测无法在tty中直接启动plasmashell）

重启方法1

首先比较直接的是直接杀死进程

$ killall plasmashell    # 杀死 ked plasma 的进程
$ kstart plasmashell    # 重新启动 plasma 桌面会话

重启方法2
直接杀进程的方法太过暴力，我们其实可以优雅的退出 palsma 之后重新启动：

$ kquitapp5 plasmashell    # 退出kde桌面
$ kstart plasmashell

重启方法3

第三种方法逼格就更高了，一句话解决问题：
alt + F2 启动搜索框，输入

$ kquitapp5 plasmashell && kstart plasmashell
